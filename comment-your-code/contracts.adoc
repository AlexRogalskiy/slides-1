=== Contracts

Usually JavaDoc on classes and methods.

They define:

* the code's central abstraction
* pre- and postconditions
* quirks

[NOTE.speaker]
--
* highlight "define" (not "describe")
* they make a promise
--

=== Analysis

Currency::
* high
Proximity::
* right on the class or method
Alternatives::
* good names (limited)
* tests (limited)
